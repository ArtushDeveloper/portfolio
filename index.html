3D Rendering Engine

I designed and programmed a fully-fledged 3D game engine based on the DirectX 11 framework using HLSL and C++. The engine is capable of mesh rendering and texturing, skeletal animation, mesh skinning, scene lighting, multiple cameras, and 2D fonts for player HUD. Additionally, I developed an offline conversion tool to process data such as mesh geometry, mesh skinning information, animation clips, and textures from .glb files into an engine-compatible format using Google Protocol Buffers.

For the high-level foundation of my engine (C++) I encapsulated complex graphics concepts, such as mesh, texture, animation clip, light, and other resources, into separate classes for improved ease of use, readability, and scalability. This design makes it straightforward to add additional graphics concepts and techniques to the engine. Additionally, I developed several helper static libraries (listed below).

For the low-level core of the engine (HLSL), I implemented over 20 DirectX shaders, including Vertex, Pixel, and Compute shaders, to manage complex scene rendering pipelines. The most prominent of these is the Volumetric Deferred Lighting Pipeline.

The full list of features:

  • Mesh Rendering
  • Mesh Texturing
  • Object World Transformations
  • Multiple Dynamic Cameras
  • Mesh Skinning and Skeleton Animation
  • Dynamic and Static Lights(Point, Spot, Directional and Area Lights)
  • Volumetric Deferred Lighting Pipeline, capable of handling up to 10000 dynamic OR static lights
  • 2D Font System
  • Helper Static Libraries(APIs):
    • Memory Management Library
    • Math Library
    • Object Library
    • File Library
    • Manager Library
    • Time Library

Part 2: Animation and Skinning

This demo shows Mesh Skinning and Skeleton Animation performed on some character models taken from an open-source website Mixamo using DirectCompute. Specifically:

  • Clip compression on the conversion tool side resulting in 3.3x less data processed at run-time, while plausibly preserving the visual animation
  • 2D Anim space blending. Blends 2 clips together over a period of time to create smoother transition between clips as opposed to switching clips instantly with no interpolation
  • Multiple animations being rendered on different skeletons and models with DirectX Compute Shaders, utilizing the GPU's multithreaded compute power (at least 1024 threads per model)

Deferred Lighting

The demo shows the power of the engine's Volumetric Deferred Lighting pipeline capable of rendering ~11000 lights before noticeable drop in frame rate as opposed to ~360 lights with Forward Lighting one. Volumetric part of the pipeline renders light volumes as Sphere (for point lights) meshes to only process pixels directly affected by a specific light source. The scene is being rendered into a temporary buffer in a Deferred fashion to be processed at a later pass
